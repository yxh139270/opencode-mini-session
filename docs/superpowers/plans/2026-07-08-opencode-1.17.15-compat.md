# OpenCode 1.17.15 Compatibility Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Upgrade the plugin to OpenCode `1.17.15`-compatible dependencies and introduce a thin compatibility layer that centralizes new/old host API differences without changing mini-session behavior.

**Architecture:** Keep business behavior in existing files, but move host-version decisions into a dedicated `src/opencode-compat.ts` layer. `src/keybinds.ts` keeps action and command builders, while `src/index.ts` delegates registration strategy to the compatibility layer.

**Tech Stack:** TypeScript, Vitest, OpenCode TUI plugin API, OpenTUI

```ts
export function registerMiniBindings(options: RegisterMiniBindingsOptions) {
  if (supportsKeymap(options.api)) {
    registerKeymapPanelLayer(...);
    registerKeymapGlobalLayer(...);
    return;
  }

  if (!options.api.command) return;
  const globalDispose = registerLegacyGlobalCommands(...);
  options.api.lifecycle.onDispose(globalDispose);
}
```
